# 하네스 엔지니어링(Harness Engineering) 통합 가이드

> AI 에이전트에게 '무엇을 할지' 지시하는 것을 넘어,  
> AI가 **최상의 성과를 낼 수 있는 '제약과 도구가 갖춰진 환경'**을 설계하는 기술

---

## 1. 하네스 엔지니어링이란?

### 어원
하네스(Harness)는 말에 씌우는 **마구(고삐, 안장)**입니다.  
아무리 힘이 센 말도 마구가 없으면 제멋대로 뜁니다.  
강력한 AI도 적절한 제약이 없으면 엉뚱한 결과를 냅니다.

### 비유
AI 모델이 **CPU**라면, 하네스는 그 CPU를 효율적으로 돌리는 **운영체제(OS)**입니다.  
아무리 좋은 CPU도 OS 없이는 제대로 쓸 수 없습니다.

### 한 줄 정의
AI 에이전트를 감싸고 있는 **제약 조건, 도구, 피드백 루프, 문서화 등 전체 운영 환경**을 설계하고 개선하는 기술.  
단순히 "프롬프트를 잘 쓰는 것"이 아니라, AI가 일하는 **사무실 전체를 설계**하는 개념입니다.

## 4. 실전 개발 워크플로우 — 5단계

인간의 개입을 최소화하고 AI가 스스로 완결성 있는 코드를 짜게 만드는 5단계 프로세스입니다.

### Step 1 — 환경 설정 `claude.md & agent.md`

- 프로젝트 루트에 `claude.md`를 두어 에이전트가 시작 시 반드시 읽게 합니다
- 기술 스택, 빌드/테스트 명령어, 상세 규칙 파일들이 어디에 있는지 경로(주소)를 적어줍니다
- 모든 내용을 한 파일에 넣지 말고 `@다른파일.md` 참조 방식으로 분리합니다

### Step 2 — 작업 격리 `Git Worktree`

- 메인 브랜치에서 직접 작업하지 않습니다
- AI가 별도의 worktree를 생성해 독립된 공간에서 구현 및 테스트를 진행합니다
- 메인 환경이 망가지는 것을 방지하고 인간의 확인 없이도 안전한 실험을 가능하게 합니다

### Step 3 — 강제성 부여 `Husky & Verify Scripts`

- `scripts/verify-task.sh` 같은 파일을 만들어 린트·단위 테스트·빌드를 한 번에 실행합니다
- Husky를 사용해 커밋 전 반드시 위 검증 스크립트를 통과해야만 저장되도록 물리적으로 강제합니다

```bash
# scripts/verify-task.sh 예시
npm run lint
npx tsc --noEmit
npm run test -- --run
npm run build
```

### Step 4 — 가시성 확보 `Logging & Screenshots`

- 테스트 실패 시 에러 로그를 `logs/` 폴더에 저장합니다
- UI 프로젝트라면 스크린샷을 `logs/screenshots/`에 저장합니다
- AI는 이 로그와 이미지를 보고 스스로 실수를 판단해 코드를 수정합니다

### Step 5 — 자동 문서화 및 머지 `Merge`

- `docs/plans/` 폴더에 작업 계획(active)과 완료 기록(completed)을 자동으로 관리합니다
- 모든 테스트가 통과되면 AI가 직접 커밋 메시지를 작성하고 메인 브랜치에 합칩니다

## 5. 5가지 핵심 원칙

| # | 원칙 | 설명 |
|---|------|------|
| 01 | 모델보다 환경 | 결과가 나쁘면 모델을 바꾸기 전 하네스 시스템부터 점검하세요. 결과물이 마음에 안 들면 프롬프트를 고치기보다 테스트 코드나 제약 조건을 하나 더 추가하세요. |
| 02 | 실패 기반 진화 | AI가 실수하면 "다음엔 잘해"라고 하지 말고, 그 실수가 구조적으로 불가능하도록 하네스를 업데이트하세요. |
| 03 | 최소 규칙의 법칙 | 규칙이 너무 많으면 AI를 혼란스럽게 합니다. 핵심적인 '지도'만 제공하고 상세 정보는 필요할 때 찾아보게 하세요. |
| 04 | 인간은 병목이다 | 인간의 테스트를 기다리게 하지 마세요. AI가 스스로 테스트하고 확신을 가질 수 있는 검증 시스템을 만드는 데 집중하세요. |
| 05 | 하네스도 코드다 | 하네스 시스템 자체도 프로젝트와 함께 성장해야 합니다. 모델 성능이 올라가면 복잡한 하네스를 단순화하는 과정이 필요합니다. |

### 기둥 03 — 피드백 루프

#### 배포 전 필수 체크 (필수)
```markdown
## 배포 전 필수 체크 (이 단계를 건너뛰지 말 것)
```

```markdown
  - npm run lint
```
